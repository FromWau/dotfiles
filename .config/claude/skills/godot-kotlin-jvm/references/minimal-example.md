# Minimal working example — Godot 4.5 + plugin 0.14.3-4.5.1

A complete, verified setup. Every file below is copy-paste ready for a fresh
project sitting next to a `project.godot` (Godot fork 4.5.x binary). Adjust
versions in `gradle/libs.versions.toml` if your fork version differs.

Project name is `my-game` for illustration — change `rootProject.name` to
match your folder.

## File tree

```
my-game/                       ← same folder as project.godot
├── project.godot              ← from the Godot editor
├── scenes/game.tscn
├── src/main/kotlin/
│   ├── Game.kt
│   └── Person.kt              ← plain helper (not registered)
├── gradle/
│   ├── libs.versions.toml
│   └── wrapper/...
├── settings.gradle.kts
├── build.gradle.kts
├── gradle.properties
├── mise.toml                  ← optional, dev convenience
└── .gitignore
```

## `gradle/libs.versions.toml`

```toml
[versions]
godot-kotlin-jvm = "0.14.3-4.5.1"
kotlinx-datetime = "0.6.2"
jdk = "17"

[libraries]
kotlinx-datetime = { module = "org.jetbrains.kotlinx:kotlinx-datetime", version.ref = "kotlinx-datetime" }

[plugins]
godot-kotlin-jvm = { id = "com.utopia-rise.godot-kotlin-jvm", version.ref = "godot-kotlin-jvm" }
```

## `settings.gradle.kts`

```kotlin
plugins {
    id("org.gradle.toolchains.foojay-resolver-convention") version "1.0.0"
}

rootProject.name = "my-game"
```

## `build.gradle.kts`

```kotlin
plugins {
    alias(libs.plugins.godot.kotlin.jvm)
}

repositories {
    mavenCentral()
}

dependencies {
    implementation(libs.kotlinx.datetime)
}

val jdkVersion = libs.versions.jdk.get().toInt()

kotlin {
    jvmToolchain(jdkVersion)
}

godot {
    registrationFileBaseDir.set(projectDir.resolve("scripts"))
    isRegistrationFileGenerationEnabled.set(true)
}

// Configure the embedded-JRE task. javaHome is a String field
val javaToolchains = extensions.getByType<JavaToolchainService>()
val jdkLauncher = javaToolchains.launcherFor {
    languageVersion.set(JavaLanguageVersion.of(jdkVersion))
}

tasks.named<godot.gradle.tasks.GenerateEmbeddedJreTask>("generateEmbeddedJre") {
    javaHome = jdkLauncher.get().metadata.installationPath.asFile.absolutePath
}
```

## `gradle.properties`

```properties
kotlin.code.style=official
```

Keep it minimal. **Do not** add `org.gradle.java.home=/abs/path` — that
hardcodes a developer-machine path and breaks for collaborators. The
toolchain + foojay handles JDK selection portably.

## `mise.toml` (optional)

```toml
[tools]
java = "temurin-17"
```

Only affects the developer's CLI shell — Gradle's internal JDK selection
goes through the toolchain mechanism, not mise. But useful so a fresh
clone + `mise install` provisions JDK 17 for `./gradlew` invocations.

## `src/main/kotlin/Game.kt` (registered Godot script)

```kotlin
import godot.annotation.RegisterClass
import godot.annotation.RegisterFunction
import godot.api.Node2D
import godot.global.GD
import kotlinx.datetime.Instant

@RegisterClass
class Game : Node2D() {
    @RegisterFunction
    override fun _ready() {
        Person(
            name = "Bill",
            bornAt = Instant.parse("1960-10-15T00:00:00Z"),
        ).also {
            GD.print("Age of ${it.name}: ${it.age}")
        }
    }
}
```

Notes:
- No `package` declaration → `.gdj` lands at `scripts/Game.gdj` (flat). Add a
  `package foo.bar` line if you want `scripts/foo/bar/Game.gdj` (hierarchy).
- `GD.print`, not `println` — only `GD.print` writes to the editor's Output
  panel.

## `src/main/kotlin/Person.kt` (plain helper — not registered)

```kotlin
import kotlinx.datetime.Clock
import kotlinx.datetime.Instant

data class Person(
    val name: String,
    val bornAt: Instant,
) {
    val age: Int
        get() {
            val days = Clock.System.now().minus(bornAt).inWholeDays
            return (days / 365).toInt()
        }
}
```

No `@RegisterClass` — this is consumed only from other Kotlin code, never
attached to a node. Registering plain data classes triggers the "no-arg
constructor required" error and produces useless `.gdj` files.

## `.gitignore`

```gitignore
# Godot 4+
.godot/
/android/

# Gradle
.gradle
build/
!gradle/wrapper/gradle-wrapper.jar

# Godot Kotlin/JVM — runtime JARs and embedded JREs, regenerated by
# ./gradlew build / generateEmbeddedJre
jvm/

# IntelliJ
.idea/
*.iml
.kotlin

# OS
.DS_Store
```

Critical exclusion: **`scripts/` is NOT ignored**. The `.gdj` files there
are referenced by `.tscn` files via `res://scripts/X.gdj` — if missing on
clone, scenes won't load Kotlin nodes properly.

## Workflow from a fresh checkout

```bash
# 1. (Optional) provision JDK 17 for the dev shell
mise install

# 2. Build — produces scripts/Game.gdj + jars under build/libs/ and jvm/
./gradlew build

# 3. (Optional) generate embedded JRE for export — silences the engine's
#    "embed a JRE with jlink" warning (can be done from godot UI)
./gradlew generateEmbeddedJre
```

Then in the Godot editor:

1. **Project → Reload Current Project** (so the editor sees new `.gdj`
   files).
2. Open `scenes/game.tscn`.
3. Select the root Node2D in the scene tree.
4. In the **Inspector**, scroll to the `Script` property → click the
   dropdown arrow → **Load** → browse to `res://scripts/Game.gdj` → Open.
   (Do NOT use the "Attach Script" right-click action — that's for creating
   *new* scripts and won't load existing `.gdj` files.)
5. Save scene (Ctrl+S).

## What you'll see on first build

```
build/libs/
  godot-bootstrap.jar         ← godot-kotlin-jvm runtime
  main.jar                    ← your compiled code
  my-game.jar                 ← shadow jar (combined)
jvm/                          ← same jars, where the engine loads from
  godot-bootstrap.jar
  main.jar
  my-game.jar
  jre-amd64-linux/            ← after generateEmbeddedJre
scripts/
  Game.gdj                    ← only @RegisterClass classes appear here
```
